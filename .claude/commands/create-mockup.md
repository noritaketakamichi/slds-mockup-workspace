# スクリーンショットからSLDSモックアップを生成

ユーザーが提供するSalesforce画面のスクリーンショットを元に、SLDSに準拠したHTMLモックアップを生成するワークフロー。

## 入力

$ARGUMENTS

引数が空の場合は、ユーザーにスクリーンショットのパスとファイル名を確認すること。

## ワークフロー

### Step 1: スクリーンショット分析

提供された画像を詳細に分析し、以下の要素を特定する:

1. **グローバルヘッダー**: アプリ名、ナビタブの一覧
2. **Highlights Panel**: オブジェクトラベル、レコード名、ハイライト項目、アクションボタン
3. **Path**: ステータスステップがあるか
4. **レコード本体**: 左カラム（タブ一覧、セクション、フィールド一覧）と右カラム（活動/Chatter/関連リスト）
5. **その他**: モーダル、トースト、フッター等

### Step 2: テンプレート参照

既存テンプレートとモックアップを読み込んで構造を参考にする:

- `.claude/templates/record-detail-page.html`（ベース構造）
- `.claude/templates/parts/` 配下のパーツ
- `mockups/user-detail.html`（最新のリファレンス実装）

### Step 3: HTML生成

以下のルールを厳守してHTMLを生成する:

#### アイコン
- **絵文字は絶対に使わない** — すべてインラインSVGで描画
- Material Icons のパスデータを活用してよい

#### ボタングループ
- Highlights Panel のアクションボタンは `gap: 0` + `margin-left: -1px` で隙間なく詰める
- 最左ボタンだけ左角丸、最右ボタンだけ右角丸

#### グローバルヘッダー
- `#032D60` 紺色バー、SVGクラウドロゴ、検索バー、"Partners" バッジ、SVGユーティリティアイコン

#### JavaScript
- 各ボタンに `id` を付与し、`buttonHandlers` オブジェクトでハンドラをまとめる
- `MockupRouter` ユーティリティを組み込む（`navigateTo`, `showModal`, `hideModal`）
- タブ切り替えは `data-tab` 属性 + イベント委譲
- セクション折りたたみは `toggleSection(id)` + `is-hidden` クラス

### Step 4: 出力と確認

1. `mockups/{機能名}-{画面種別}.html` に保存
2. `open` コマンドでブラウザ表示
3. 元画像との差異がないか確認し、差異があればリストアップして報告

## チェックリスト

生成後、以下を確認:

- [ ] SLDSのCDNが読み込まれているか
- [ ] 絵文字が一切使われていないか（全SVGか）
- [ ] ボタングループが隙間なく詰まっているか
- [ ] タブ切り替え・セクション折りたたみが動作するか
- [ ] `buttonHandlers` と `MockupRouter` が組み込まれているか
- [ ] フォームフィールドに編集ペンシルアイコン（SVG）があるか
