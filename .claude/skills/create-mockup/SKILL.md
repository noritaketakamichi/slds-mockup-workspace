# Create Mockup

スクリーンショットを元にSLDS準拠のHTMLモックアップを生成するワークフロー。

## ワークフロー

### Step 1: スクリーンショット分析

提供された画像を詳細に分析し、以下の要素を特定する:

1. **グローバルヘッダー**: アプリ名、ナビタブの一覧
2. **Highlights Panel**: オブジェクトラベル、レコード名、ハイライト項目、アクションボタン
3. **Path**: ステータスステップがあるか
4. **レコード本体**: 左カラム（タブ一覧、セクション、フィールド一覧）と右カラム（活動/Chatter/関連リスト）
5. **その他**: モーダル、トースト、フッター等

引数が空の場合は、ユーザーにスクリーンショットのパスとファイル名を確認する。

### Step 2: テンプレート参照

既存テンプレートとモックアップを読み込んで構造を参考にする:

- `.claude/templates/record-detail-page.html`（ベース構造）
- `.claude/templates/parts/` 配下のパーツ
- `mockups/user-detail.html`（リファレンス実装）

### Step 3: HTML生成

以下のルールを厳守:

- **アイコン**: 絵文字は絶対に使わない。すべてインラインSVG。Material Iconsのpathデータ活用可
- **ボタングループ**: `gap: 0` + `margin-left: -1px` で隙間なく詰める。最左だけ左角丸、最右だけ右角丸
- **グローバルヘッダー**: `#032D60` 紺色バー、SVGクラウドロゴ、検索バー、"Partners" バッジ
- **JavaScript**: `buttonHandlers` オブジェクト、`MockupRouter` ユーティリティ、`data-tab` タブ切り替え、`toggleSection(id)` 折りたたみ

### Step 4: 出力と確認

1. `mockups/{機能名}-{画面種別}.html` に保存
2. 元画像との差異がないか確認し、差異があればリストアップして報告

## チェックリスト

- SLDSのCDNが読み込まれているか
- 絵文字が一切使われていないか（全SVGか）
- ボタングループが隙間なく詰まっているか
- タブ切り替え・セクション折りたたみが動作するか
- `buttonHandlers` と `MockupRouter` が組み込まれているか
